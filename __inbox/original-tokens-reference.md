## Anomalies for downstream waves

1. .katex .base uses --zd-fg directly (not --color-fg). This is the only place
   in global.css where a --zd-* raw variable is used directly in a non-@theme
   context. All other consumer code uses --color-* aliases.

2. The "Default Light" scheme uses palette index 5 (#a35e0f — a brownish amber)
   as the accent, which is an unusual semantic choice for an accent color.
   The "Default Dark" scheme uses palette index 12 (#d69a66 — warm orange) for
   accent, not index 5 (#c074d6 — purple), which would be the slot-convention
   default. Both are explicit overrides in scheme.semantic.

3. p13 in "Default Dark" is the same hex (#c074d6) as p5 in that scheme. This
   means --zd-13 and --zd-5 are identical in Default Dark.

4. chatUserBg in both schemes resolves to the accent slot (default index 5),
   meaning chat user bubbles will have the accent color as background. This is
   intentional per the semantic defaults in color-scheme-utils.ts.

5. No component has a <style> block with design tokens except page-loading-overlay.astro.
   All other components use Tailwind utility classes from @theme. There are no
   component-specific CSS custom properties.

6. The version-banner.astro uses Tailwind's opacity modifier syntax
   (border-warning/30, bg-warning/5) for semi-transparent admonition backgrounds.
   This means --color-warning is used at 30% and 5% opacity rather than via
   color-mix — a different pattern from admonition.astro which uses color-mix.
